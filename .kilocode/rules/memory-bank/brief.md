This project is a performance-optimized, multi-threaded solver for hexagonal "Lights Out" puzzles, identified as PDC-2018Q35 (named after the 2018 Pi Day Challenge's Question 35, which inspired this program). Written in Java, it employs a parallel brute-force approach to find solutions for various grid configurations, written with the express purpose of using it to solve Q35 (a puzzle that has stumped myself for 7 months). Since this puzzle is the most difficult of the hexagonal puzzles featured in the challenge, it is important to optimize this solver for maximum performance.

The architecture is built on a producer-consumer model, utilizing a `ForkJoinPool` for efficient, recursive generation of click combinations and a pool of worker threads to validate them. Key technologies include Java 25, Maven for build management, and Log4j2 for asynchronous logging. The primary objective is to solve computationally intensive puzzles by leveraging modern multi-core processors effectively.
